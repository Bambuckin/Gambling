# Milestone v1.1: Domain Consolidation and Truth Model

**Status:** SHIPPED 2026-04-22
**Audit:** gaps_found (.planning/v1.1-MILESTONE-AUDIT.md)
**Phases:** 1-25
**Total Plans:** 59

## Overview

This milestone shipped the full local lottery terminal operations contour and then consolidated runtime truth onto canonical purchase, draw, and purchase_attempt models without breaking the working Big 8 cashier/admin flow.

## Key Accomplishments

1. Delivered the shared web/admin/runtime platform with login, registry, ledger, queue orchestration, audit, and module-level documentation.
2. Landed a truthful Big 8 operator contour: live draw sync, terminal-driven purchase progression, manual draw close/settlement, notifications, and explicit winning actions.
3. Migrated runtime truth onto canonical purchase, draw, and purchase_attempt storage while keeping current surfaces alive through compatibility projections.
4. Rebased winnings, admin, receiver, and user read models onto canonical-first projections and explicit fulfillment jobs.
5. Replaced TTL execution-lock semantics with advisory locking and introduced a replaceable queue transport seam.
6. Demoted active dependency on legacy ticket writes and validated the canonical-first purchase -> draw -> winnings contour.

## Known Gaps Accepted At Archive Time

- Formal VERIFICATION.md artifacts exist only for phases 18-21; phases 1-17 and 22-25 are covered by summaries and validation history, not by full retrospective phase verification docs.
- Live target-LAN NLoto selector/session smoke is still outstanding.
- Physical compatibility-table cleanup remains deferred to a deliberate migration window.
- Nyquist validation artifacts are absent for the milestone.
